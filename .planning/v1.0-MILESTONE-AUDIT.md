### Integration Notes (Non-Blocking)

**validate-scenario missing cluster guard (minor):** `_cmd_validate_scenario` does not call `cluster_check_active` before kubectl operations. When no cluster is running, raw kubectl errors appear instead of the friendly "No active cluster" message. The `drill` and `next` subcommands guard correctly. This is a UX polish item, not a functional break — validate-scenario still fails correctly, just with ugly output. Affects DIST-03/DIST-04 cosmetically only.

**Exam history empty in drill mode (by design):** `progress_read_exam_history` returns `[]` because no exam writer exists yet. The status display correctly gates on `exam_count > 0` and skips the section. The writer is deferred to Phase 4 (EXAM-11). Not a wiring break.
