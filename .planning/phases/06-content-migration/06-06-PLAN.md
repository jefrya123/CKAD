<task id="2" type="auto">
<title>Fix placeholder in debug-volume-mount.yaml solution step 4</title>
<files>

<action>
In `scenarios/domain-1/debug-volume-mount.yaml`, solution step 4 (the last step, line ~73) currently reads:
```yaml
- "kubectl delete pod lab03-pod -n debug-lab03 && kubectl apply -f <the-fixed-manifest>"
```

This is a placeholder — `<the-fixed-manifest>` is not valid shell. The solution step 3 already contains the complete corrected pod manifest applied via heredoc (`kubectl apply -f - <<EOF ... EOF`). Step 4 is therefore redundant — the delete+recreate is already handled by step 3's apply (which replaces the pod).

**Fix:** Remove solution step 4 entirely (delete the line). Step 3 already applies the fixed manifest. The pod is immutable (volumeMounts change requires recreation), but `kubectl apply` will error — so change step 3's approach to delete-then-apply:

Replace the entire solution.steps with:
```yaml
solution:
  steps:
    - "kubectl describe pod lab03-pod -n debug-lab03 | grep -A5 Mounts"
    - "kubectl get pod lab03-pod -n debug-lab03 -o yaml | grep -A5 volumes"
    - "kubectl delete pod lab03-pod -n debug-lab03"
    - |
      kubectl apply -f - <<EOF
      apiVersion: v1
      kind: Pod
      metadata:
        name: lab03-pod
        namespace: debug-lab03
      spec:
        containers:
        - name: app
          image: busybox
          command: ["sh", "-c", "cat /config/app.conf; sleep 3600"]
          volumeMounts:
          - name: config
            mountPath: /config
        volumes:
        - name: config
          configMap:
            name: lab03-config
      EOF
```

This separates delete (step 3) and apply (step 4) into distinct steps, both valid shell commands. The old placeholder step is removed.
</action>
