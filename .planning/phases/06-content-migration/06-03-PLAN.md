Example debug scenario structure:
```yaml
id: debug-wrong-image
domain: 3
title: "Fix: Wrong Image Name"
difficulty: easy
time_limit: 120
namespace: debug-ns
description: >
  A pod has been deployed but is stuck in ImagePullBackOff.
  Diagnose the issue and fix the pod so it runs successfully.
setup:
  manifest: |
    apiVersion: v1
    kind: Pod
    metadata:
      name: lab01-web
      namespace: debug-ns
    spec:
      containers:
      - name: web
        image: ngnix   # <-- the bug
        ports:
        - containerPort: 80
hint: >
  Check kubectl describe pod to see the image pull error.
validations:
  - name: pod_running
    type: container_running
    pod: lab01-web
    namespace: debug-ns
solution:
  steps:
    - "kubectl get pod lab01-web -n debug-ns"
    - "kubectl describe pod lab01-web -n debug-ns | grep -A3 Events"
    - "kubectl edit pod lab01-web -n debug-ns  # Fix image: ngnix -> nginx"
```
