    - learn_show_intro on file with no learn_intro outputs empty string

  <action>
Create lib/learn.sh (sourced library, no shebang, no set strict mode — same pattern as all other lib/*.sh files). Header comment: `# lib/learn.sh — learn mode discovery, ordering, and lesson progression`. Add `# shellcheck shell=bash` and `# shellcheck disable=SC2034`.

Functions to implement:

1. `learn_discover [EXTERNAL_PATH]` — Calls `scenario_discover "$external_path"`, then filters to only files where `yq -r '.learn_intro // empty' "$file"` is non-empty. Outputs matching file paths. This is the learn-mode subset of all scenarios.

2. `_learn_difficulty_rank DIFFICULTY` — Internal helper. Maps easy=1, medium=2, hard=3. Used for sort ordering.

3. `_learn_sort_files FILE...` — Internal helper. Takes file paths as args, outputs them sorted by domain (ascending) then difficulty rank (ascending). Uses a decorated-sort pattern: for each file, extract domain and difficulty via yq, prepend "DOMAIN RANK FILE", sort numerically, then strip prefix. This gives progressive ordering within each domain.

4. `learn_list_domain DOMAIN` — Discovers learn scenarios, filters to given domain (use FILTER_DOMAIN with scenario_filter, or manual yq check), sorts progressively, then for each: checks completion via `progress_learn_completed "$id"` and prints `[x] $title` or `[ ] $title`. Returns nothing if no learn scenarios exist for that domain.

5. `learn_show_intro FILE` — Extracts and outputs `yq -r '.learn_intro // empty' "$file"`. Simple wrapper for consistent API.

6. `learn_next_lesson DOMAIN` — Discovers + sorts learn scenarios for given domain. Iterates in order, returns the first file path where `progress_learn_completed "$id"` returns 1 (not completed). Outputs empty string if all are done.

Create test/unit/learn.bats with test fixtures. Tests should create temp scenario YAML files with and without learn_intro fields. Stub progress functions as needed (or use real progress.json in temp dirs). Follow existing test/unit/*.bats patterns for setup/teardown.
