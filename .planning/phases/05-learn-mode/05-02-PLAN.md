Add learn mode to bin/ckad-drill. Follow established patterns from exam mode wiring (Phase 4 Plan 02).

1. **Source lib/learn.sh** — Add `source "${CKAD_DRILL_ROOT}/lib/learn.sh"` after the existing lib sources.

2. **Create `_learn_start DOMAIN [EXTERNAL_PATH]` helper function** (same pattern as _exam_start/_drill_start):
   - Call `learn_next_lesson "$domain"` to get the next uncompleted lesson file
   - If empty (all done): print success message "All lessons in Domain $domain complete!" and return
   - Call `cluster_check_active || exit "${EXIT_NO_CLUSTER}"`
   - Install cleanup trap: `trap '_drill_cleanup' INT TERM EXIT`
   - Call `scenario_setup "$lesson_file"`
   - Write session: `session_write "learn" "${SCENARIO_ID}" "${SCENARIO_FILE}" "${SCENARIO_NAMESPACE}" "${SCENARIO_TIME_LIMIT}"`
   - Display concept text: call `learn_show_intro "$lesson_file"` — print with a "Concepts" header if non-empty
   - Display the scenario task card: `_drill_display "$lesson_file"`
   - Info line: "Run: ckad-drill check when ready. Hints and solutions are available."
   - Remove EXIT trap (session established)

3. **Create `_learn_list` helper function:**
   - For each domain 1-5: print domain header, call `learn_list_domain "$domain"` to show "[x]/[ ] title" lines
   - If a domain has no learn scenarios, print "No lessons available yet"
   - At bottom, print: "Run: ckad-drill learn --domain N to start a lesson"

4. **Add `learn)` case to subcommand dispatch:**
   ```
   learn)
     shift
     domain_filter=""
     while [[ $# -gt 0 ]]; do
       case "$1" in
         --domain) domain_filter="$2"; shift 2 ;;
         *) error "Unknown flag: $1"; exit "${EXIT_ERROR}" ;;
       esac
     done
     if [[ -n "${domain_filter}" ]]; then
       _learn_start "${domain_filter}" ""
     else
       _learn_list
     fi
     ;;
   ```

5. **Extend `check)` for learn mode** — Add `elif [[ "${SESSION_MODE}" == "learn" ]]` branch after the exam and drill branches:
   - Run validator: `validator_run_checks "${SESSION_SCENARIO_FILE}" "${SESSION_NAMESPACE}" || validator_exit=$?`
   - If passed: call `progress_record_learn "${SESSION_SCENARIO_ID}"`, print success, then call `learn_next_lesson` for the scenario's domain to suggest next lesson (LERN-04). If next lesson exists, print "Next lesson: $title. Run: ckad-drill learn --domain $domain". If no next, print "All lessons in Domain $domain complete!"
   - If failed: print "Not yet correct. Keep trying! Hints and solutions are available."
   - Also record drill-style progress via `progress_record` so learn attempts appear in status (reuses existing domain logic)

6. **Update help text** — Add learn subcommands between drill and exam sections:
   ```
   info "  learn                  List all domains with lesson completion status"
   info "  learn --domain N       Start the next lesson in domain N"
   ```

7. **Write tests in test/unit/drill.bats** — Append learn mode tests following existing test patterns:
   - Test: `ckad-drill learn` without session prints domain listing (mock learn_discover/learn_list_domain)
   - Test: learn mode check branch (mock session with mode=learn)
   - Test: hint allowed in learn mode (not blocked like exam)
   - Test: solution allowed in learn mode (not blocked like exam)

IMPORTANT patterns to follow:
