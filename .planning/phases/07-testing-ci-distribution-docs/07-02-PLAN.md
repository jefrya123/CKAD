  <action>
Create scripts/install.sh — a self-contained install script that can be run via `curl -sSL URL | sh`.

**Requirements (DIST-01):**

1. **Detect OS and architecture** — `uname -s` (Linux/Darwin), `uname -m` (x86_64/arm64/aarch64). Map to download URLs for kind, yq, jq.

2. **Check required dependencies** that cannot be installed:
   - Docker: `command -v docker` — if missing, print install instructions and exit
   - kubectl: `command -v kubectl` — if missing, print install instructions and exit
   - bash >= 4.0: Check `${BASH_VERSINFO[0]}` — if < 4, print upgrade instructions and exit

3. **Install optional dependencies** if missing:
   - kind: Download from `https://kind.sigs.k8s.io/dl/v0.25.0/kind-{os}-{arch}` to `~/.local/bin/kind`
   - yq: Download from `https://github.com/mikefarah/yq/releases/download/v4.44.6/yq_{os}_{arch}` to `~/.local/bin/yq` (note: project uses yq v3 SYNTAX but yq v4 binary is backward compatible via `-r` flag)
   - jq: Download from `https://github.com/jqlang/jq/releases/download/jq-1.7.1/jq-{os}-{arch}` to `~/.local/bin/jq`

4. **Install ckad-drill itself:**
   - Clone repo to `~/.local/share/ckad-drill` (or use CKAD_DRILL_HOME env var)
   - Create symlink `~/.local/bin/ckad-drill` -> `~/.local/share/ckad-drill/bin/ckad-drill`
   - Ensure `~/.local/bin` is in PATH — print a warning if not

5. **Script conventions:**
   - `set -euo pipefail`
   - Use colors (green check, red x, yellow warning) but detect if stdout is a terminal first
   - Print what is being done at each step
   - Idempotent — running twice should work without error
   - No `sudo` unless absolutely necessary (prefer ~/.local/bin)
   - Set GITHUB_REPO variable at top for easy customization (default: user/ckad-drill placeholder — the actual repo URL will be set when published)

6. **Do NOT use `sudo` for any install step.** Use `~/.local/bin` for all binaries. If `~/.local/bin` doesn't exist, create it.

The script must pass shellcheck.
