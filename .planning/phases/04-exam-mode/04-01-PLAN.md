  <name>Task 1: Create lib/exam.sh with exam session engine</name>
  <files>lib/exam.sh, test/unit/exam.bats</files>
  <behavior>
    - exam_select_questions: Given a pool of scenario files and a target count (default 16), selects questions weighted by CKAD domain percentages (D1:20%, D2:20%, D3:15%, D4:25%, D5:20%). If a domain has fewer scenarios than its target, fills from other domains. Returns file paths.
    - exam_session_write: Creates session.json with mode="exam", a questions array (each entry: {id, file, namespace, domain, status:"pending", flagged:false}), current_question index (0-based), global end_at epoch. Uses jq atomic write (tmp+mv).
    - exam_session_read: Reads exam session.json, populates EXAM_QUESTIONS (JSON), EXAM_CURRENT (index), EXAM_END_AT (epoch), SESSION_MODE. Returns EXIT_NO_SESSION if missing or mode != exam.
    - exam_list: Reads questions array, outputs formatted list with index, title, status icon (pending=[ ], passed=[+], failed=[x], flagged=[?]), and domain. Current question marked with >.
    - exam_navigate DIRECTION_OR_INDEX: "next" increments current, "prev" decrements, integer jumps to that 1-based index. Wraps or clamps at bounds. Updates session.json.
    - exam_flag: Toggles flagged field on current question. Updates session.json.
    - exam_grade: Iterates questions, computes per-domain pass counts and totals, overall score as percentage, returns pass (>=66%) or fail. Outputs JSON result: {score, passed, domains: [{domain, passed, total, percent}], questions: [{id, passed, flagged}]}.
    - exam_current_question: Outputs the file path and namespace of the current question (from index).
  </behavior>
  <action>
  Create `lib/exam.sh` with the following functions. File header follows existing pattern: no shebang, no strict mode, shellcheck shell=bash, SC2034 disable for globals.

  **CKAD domain weights** (target counts for 16 questions):
  - Domain 1: 3 questions (20% of 16 ≈ 3.2)
  - Domain 2: 3 questions (20%)
  - Domain 3: 2 questions (15% of 16 ≈ 2.4)
  - Domain 4: 4 questions (25% of 16 = 4)
  - Domain 5: 3 questions (20%)
  - Remaining: 1 distributed to smallest domain (D3)
  Total: 16

  **exam_select_questions COUNT [EXTERNAL_PATH]:**
  1. Call scenario_discover to get all available scenario files
  2. For each file, call scenario_load to get domain (yq -r '.domain // empty' "$file" is faster — use that instead of full scenario_load)
  3. Group files by domain into associative arrays
  4. Compute target count per domain based on CKAD weights (use integer math: count * weight / 100, distribute remainder to D4 first then D3)
  5. For each domain, shuffle and pick min(target, available) files
  6. If total < COUNT after first pass, fill from domains with remaining scenarios
  7. Shuffle final selection (not sorted by domain)
  8. Output file paths one per line

  **exam_session_write END_AT QUESTION_FILES...:**
  1. Build JSON questions array using jq: for each file, extract id/namespace/domain via yq
  2. Write session.json with: mode="exam", questions array, current_question=0, started_at (UTC ISO), end_at (epoch), time_limit
  3. Use jq atomic write pattern (tmp + mv)

  **exam_session_read:**
  1. Check CKAD_SESSION_FILE exists, return EXIT_NO_SESSION if not
  2. Read mode — if not "exam", return EXIT_NO_SESSION
  3. Set globals: EXAM_QUESTIONS (raw JSON array), EXAM_CURRENT (integer), EXAM_END_AT (epoch), EXAM_QUESTION_COUNT (integer), SESSION_MODE="exam"
  4. Set EXAM_CURRENT_FILE and EXAM_CURRENT_NAMESPACE from questions[current_question]

  **exam_update_question_status INDEX STATUS:**
  1. Read session file, update .questions[INDEX].status to STATUS
  2. Atomic write

  **exam_list:**
  1. Call exam_session_read
  2. For each question: print index (1-based), status icon, title (from yq on file), domain, flagged marker
  3. Mark current question with ">"

  **exam_navigate TARGET:**
  1. Call exam_session_read
  2. Parse TARGET: "next" → current+1, "prev" → current-1, integer → TARGET-1 (convert to 0-based)
  3. Clamp to [0, question_count-1]
  4. Update .current_question in session.json (atomic write)
  5. Set EXAM_CURRENT_FILE and EXAM_CURRENT_NAMESPACE from new index

  **exam_flag:**
  1. Call exam_session_read
  2. Toggle .questions[current_question].flagged (true↔false)
  3. Atomic write

  **exam_grade:**
  1. Call exam_session_read
  2. For each question, read status (passed/failed/pending — pending counts as failed)
  3. Compute per-domain: {domain, passed_count, total_count, percent}
  4. Compute overall: total_passed / total_questions * 100
  5. Determine pass: overall >= 66
  6. Output JSON result to stdout

  **exam_current_question:**
  1. Call exam_session_read
  2. Print EXAM_CURRENT_FILE

  **exam_setup_all_namespaces:**
  1. Call exam_session_read
  2. For each question, extract namespace and run `kubectl create namespace NS 2>/dev/null || true`
  3. Print count of namespaces created

  **exam_cleanup_all_namespaces:**
  1. Call exam_session_read (or read directly if session might be partial)
  2. For each question namespace, run `kubectl delete namespace NS --ignore-not-found=true`
  3. Call session_clear

  **Known pitfalls (from v1.0):**
  - Use `((n++)) || true` for arithmetic increments from 0 under set -e
  - Use yq v3 syntax: `yq -r '.field // empty' file`
  - Use jq atomic write: `jq ... file > tmp && mv tmp file`
  - error() is print-only, does NOT exit — caller decides
