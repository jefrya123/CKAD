<task type="auto">
  <name>Task 1: Add status and validate-scenario subcommands to bin/ckad-drill</name>
  <files>bin/ckad-drill</files>
  <action>
    Add two new cases to the bin/ckad-drill dispatch:

    **status)** (PROG-02)
    1. progress_init (ensures file exists)
    2. header "Drill Progress"
    3. Call progress_read_domain_rates and format as table:
       ```
       Domain  Pass Rate  Attempts
       1       75%        4
       2       50%        2
       3       --         0
       4       100%       1
       5       --         0
       ```
    4. Print streak: "Current streak: N day(s)" via progress_read_streak
    5. Print exam history summary via progress_read_exam_history (if any exams recorded)
    6. Print recommendation: "Weak area: Domain N — focus your next drills here" via progress_recommend_weak_domain
    7. If no scenarios attempted yet, print "No drill results yet. Run 'ckad-drill drill' to get started."

    **validate-scenario)** (DIST-03, DIST-04)
    1. Require a target argument: `local target="${2:-}"` — error if empty
    2. If target is a directory (DIST-04):
       - Find all .yaml files: `find "$target" -name "*.yaml" -type f`
       - Run _validate_single_scenario for each
       - Track pass/fail counts
       - Print summary: "Validation complete: N passed, M failed of T scenarios"
    3. If target is a file (DIST-03):
       - Run _validate_single_scenario "$target"
    4. Exit with 0 if all passed, 1 if any failed

    **_validate_single_scenario FILE** helper function:
    1. Print "Validating: $file..."
    2. scenario_load "$file" || { fail "parse" "valid YAML" "parse error"; return 1; }
    3. scenario_setup "$file" || { fail "setup" "namespace created" "setup failed"; return 1; }
    4. Set cleanup_needed=true
    5. Apply solution: set namespace context with `kubectl config set-context --current --namespace="$SCENARIO_NAMESPACE"`, then execute each solution step via eval (same pattern as validator.sh command_output). Use `yq -r '.solution.steps[]' "$file"` to get steps.
    6. validator_run_checks "$file" "$SCENARIO_NAMESPACE"; capture result
    7. scenario_cleanup (ALWAYS — even on failure per Pitfall from research)
    8. Reset namespace context: `kubectl config set-context --current --namespace=default`
    9. Print PASS or FAIL for the scenario

    Use explicit cleanup pattern (not just trap) since multiple scenarios run in sequence for directory mode. The research doc's _validate_single_scenario pattern with cleanup_needed flag is the right approach.

    IMPORTANT: yq v3 syntax. Namespace context set/reset around solution apply (Pitfall 8 from research). Always clean up even on failure.
