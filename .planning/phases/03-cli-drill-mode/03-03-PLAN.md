<task type="auto">
  <name>Task 2: Expand bin/ckad-drill with all drill subcommands</name>
  <files>bin/ckad-drill</files>
  <action>
    Expand the case dispatch in bin/ckad-drill to add 9 new subcommands. Source the 3 new lib files (session.sh, progress.sh, timer.sh) at the top alongside existing sources.

    Add these cases to the dispatch:

    **drill)** (DRIL-01, DRIL-02)
    1. Check cluster is active (cluster_check_active or similar)
    2. Parse flags: shift past "drill", loop to consume --domain N, --difficulty LEVEL, --external PATH
    3. scenario_discover [external_path] into array (mapfile from command output)
    4. Set FILTER_DOMAIN and FILTER_DIFFICULTY if provided, then scenario_filter on the array
    5. Check filtered array not empty (Pitfall 7: "No scenarios found matching filters")
    6. Pick random: `${filtered[RANDOM % ${#filtered[@]}]}`
    7. scenario_setup "$picked_file" (loads SCENARIO_* globals + creates namespace)
    8. session_write "drill" "$SCENARIO_ID" "$SCENARIO_FILE" "$SCENARIO_NAMESPACE" "$SCENARIO_TIME_LIMIT"
    9. Display: header with title, info lines for domain/difficulty/time_limit, then description via `yq -r '.description' "$picked_file"`
    10. Print: "Run: source <(ckad-drill env) to start timer and set up exam environment"
    11. Install SIGINT/SIGTERM trap (DRIL-11): trap '_drill_cleanup' INT TERM; _drill_cleanup checks session file exists, calls scenario_cleanup + session_clear. Use Pitfall 5 pattern from research.

    **check)** (DRIL-03)
    1. session_require
    2. Verify SESSION_MODE == "drill" (not exam — Phase 4 concern, but guard now)
    3. validator_run_checks "$SESSION_SCENARIO_FILE" "$SESSION_NAMESPACE"
    4. Capture exit code ($?) as passed (0=true, else false)
    5. Compute elapsed: $(( $(date +%s) - SESSION_END_AT + SESSION_TIME_LIMIT )) or store start epoch in session
    6. progress_record "$SESSION_SCENARIO_ID" "$(yq -r '.domain' "$SESSION_SCENARIO_FILE")" "$passed" "$elapsed"

    **hint)** (DRIL-04)
    1. session_require
    2. `yq -r '.hint // "No hint available."' "$SESSION_SCENARIO_FILE"`

    **solution)** (DRIL-05)
    1. session_require
    2. Display solution steps: `yq -r '.solution.steps[]' "$SESSION_SCENARIO_FILE"` with numbered formatting

    **current)** (DRIL-08)
    1. session_require
    2. Redisplay: header + domain/difficulty/time + description (same as drill display)

    **next)** (DRIL-06)
    1. session_require + session_read
    2. scenario_cleanup (deletes namespace)
    3. session_clear
    4. Re-execute drill logic (call the drill function or re-invoke)

    **skip)** (DRIL-07)
    1. session_require + session_read
    2. scenario_cleanup
    3. session_clear
    4. info "Skipped. Run 'ckad-drill drill' for next scenario."

    **env)** (TIMR-01, TIMR-02, TIMR-03)
    1. If "$2" == "--reset": timer_env_reset_output; exit
    2. session_require
    3. timer_env_output "$SESSION_END_AT"

    **timer)** (TIMR-04)
    1. session_require
    2. timer_remaining

    Update the usage/help text in the default (*) case to list all available commands.

    Extract drill display and drill flow into helper functions (_drill_display, _drill_start) so `next` can reuse them without code duplication.

    IMPORTANT: Use yq v3 syntax (yq -r '.field // empty' file). Use ((n++)) || true for arithmetic. The trap handler must be robust per Pitfall 5 from research (check session file exists before cleanup).
