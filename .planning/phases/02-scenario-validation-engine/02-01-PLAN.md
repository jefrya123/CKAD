  <action>
**display.sh** — Replace the Phase 1 stubs with working implementations. Follow the exact pattern from 02-RESEARCH.md Pattern 7:

- `pass CHECK_NAME` — prints `  [PASS] check_name` in green (uses `_color_enabled` from common.sh)
- `fail CHECK_NAME EXPECTED ACTUAL` — prints `  [FAIL] check_name` in red, then `expected:` and `actual:` lines indented below
- `header TEXT` — prints bold text with a dash underline of matching length

Important: display.sh depends on `_color_enabled()` from common.sh being already sourced. Add a comment noting this dependency. Do NOT redefine `_color_enabled`.

**scenario.sh** — Create new file following 02-RESEARCH.md Patterns 1-4 and the complete structure from Pattern "Complete scenario.sh Structure". Use `# shellcheck shell=bash` directive (no shebang, no set strict mode — sourced file per Phase 1 pattern).

Public functions to implement:

1. `scenario_discover [EXTERNAL_PATH]` (SCEN-03, SCEN-04, SCEN-05):
   - Find all `*.yaml` files under `${CKAD_DRILL_ROOT}/scenarios` using `find ... -print0 | sort -z`
   - If EXTERNAL_PATH provided and is a directory, also find YAML files there
   - Track seen IDs in an array; warn on duplicates (first-loaded wins, SCEN-05)
   - Output one file path per line for valid files
   - Use `_scenario_register_file` helper with nameref array

2. `scenario_load FILE` (SCEN-01):
   - Extract required fields via yq v3 syntax: `yq -r '.field // empty' file`
   - Required fields: id, domain, title, difficulty, time_limit
   - CRITICAL: Use yq v3 syntax (`yq -r '.field' file`), NOT yq v4 (`yq eval`)
   - Validate all required fields present; return EXIT_PARSE_ERROR if any missing
   - Namespace: use `.namespace // empty`, fallback to `drill-${id}` (ADR-06)
   - Check for helm tag: `yq -r '.tags // [] | .[]' file | grep -q "^helm$"`
   - Export to globals: SCENARIO_ID, SCENARIO_DOMAIN, SCENARIO_TITLE, SCENARIO_DIFFICULTY, SCENARIO_TIME_LIMIT, SCENARIO_NAMESPACE, SCENARIO_HAS_HELM, SCENARIO_FILE

3. `scenario_filter FILE...` (SCEN-03):
   - Accept files as arguments
   - Read FILTER_DOMAIN and FILTER_DIFFICULTY environment variables
   - For each file, extract domain and difficulty via yq; skip non-matching
   - Output matching file paths

4. `scenario_setup FILE` (SCEN-02, SCEN-06):
   - Call `scenario_load` to populate globals
   - If SCENARIO_HAS_HELM is true, check `command -v helm` — error with install URL if missing
   - Create namespace with `kubectl create namespace ... 2>/dev/null || true`
   - Process setup.commands if present (for debug scenarios)
   - Handle `kubectl apply -f -` by piping setup.manifest

5. `scenario_cleanup` (SCEN-02):
   - Delete SCENARIO_NAMESPACE with `kubectl delete namespace ... --ignore-not-found=true`
   - Safe to call even if setup was partial

Anti-patterns to avoid:
- Do NOT use `yq eval` (v4 syntax) — machine has yq v3.4.3
- Do NOT call `exit` in lib functions — return non-zero, caller decides
- Use `// empty` with yq to avoid getting the string "null" for missing fields
