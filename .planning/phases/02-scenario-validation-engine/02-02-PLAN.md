**lib/validator.sh** — Implement following 02-RESEARCH.md Patterns 5-6 exactly:

- `# shellcheck shell=bash` directive (no shebang, sourced file)
- `validator_run_checks FILE NAMESPACE` — main dispatch function:
  - Count validations with `yq -r '.validations | length' file`
  - Loop 0 to count-1, extract check_type with `yq -r ".validations[${i}].type" file`
  - Case dispatch to `_validator_*` functions
  - Each returns "PASS" or "FAIL:expected:actual"
  - Call `pass()` or `fail()` from display.sh
  - Track total/passed/failed counts
  - Print results header at end
  - Return 0 if all pass, 1 if any fail

10 private check functions (`_validator_*`):

1. `_validator_resource_exists` — `kubectl get resource -n ns` exit code
2. `_validator_resource_field` — `kubectl get resource -n ns -o jsonpath=path`, compare to expected
3. `_validator_container_count` — `kubectl get resource -o json | jq '.spec.containers | length'`
4. `_validator_container_image` — `kubectl get -o json | jq --arg name container '.spec.containers[] | select(.name==$name) | .image'`
5. `_validator_container_env` — `kubectl get -o json | jq --arg cname container --arg ename env_name '.spec.containers[] | select(.name==$cname) | .env[] | select(.name==$ename) | .value'`
6. `_validator_volume_mount` — `kubectl get -o json | jq --arg cname container --arg mpath mount_path '.spec.containers[] | select(.name==$cname) | .volumeMounts[] | select(.mountPath==$mpath) | .name'`
7. `_validator_container_running` — `kubectl get -o json | jq --arg cname container '.status.containerStatuses // [] | .[] | select(.name==$cname) | .state | has("running")'` (use `// []` fallback per Pitfall 3)
8. `_validator_label_selector` — `kubectl get resource_kind -n ns -l selector --no-headers | grep -q .`
9. `_validator_resource_count` — `kubectl get resource_kind -n ns -l selector --no-headers | wc -l` (handle empty output = 0 per Pitfall 2)
10. `_validator_command_output` — `eval command` then check with grep -qF (contains) / grep -qE (matches) / `[[ == ]]` (equals)

CRITICAL per ADR-07: No retry, no kubectl wait, no loops. Each check invokes kubectl exactly once.

All field extraction uses yq v3 syntax: `yq -r ".validations[${idx}].field" "${file}"`

Anti-patterns:
- Do NOT use `yq eval` (v4 syntax)
- Do NOT retry on failure (ADR-07)
- Do NOT call exit in lib functions
- Handle empty kubectl output in resource_count (empty string before wc -l = 0)
- Use `// []` in jq for containerStatuses that may be null
