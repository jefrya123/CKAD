<task type="auto">
  <name>Task 2: Implement cluster lifecycle in cluster.sh</name>
  <files>lib/cluster.sh</files>
  <action>
Create `lib/cluster.sh` with the full cluster lifecycle. This is sourced by bin/ckad-drill — do NOT add shebang or set strict mode.

**Addon version variables at the top** (per CONTEXT decision — dev-friendly, not user-configurable):
```
CALICO_VERSION="v3.31.4"
INGRESS_NGINX_VERSION="v1.14.3"
METRICS_SERVER_VERSION="v0.8.1"
```

**Dependency checking — `cluster_check_deps()`:**
- Loop over `docker kind kubectl yq jq` checking `command -v`
- Docker special case: if binary exists, also check `docker info &>/dev/null 2>&1` — if daemon not running, add "docker (installed but daemon not running)"
- Accumulate ALL missing deps in an array (do not fail-on-first, per CONTEXT decision)
- If any missing: print each missing dep, then print install instructions with BOTH install command AND docs URL for each:
  - docker: `https://docs.docker.com/get-docker/`
  - kind: `https://kind.sigs.k8s.io/docs/user/quick-start/#installation`
  - kubectl: `https://kubernetes.io/docs/tasks/tools/`
  - yq: `https://github.com/mikefarah/yq#install`
  - jq: `https://jqlang.github.io/jq/download/`
- Return 1 if any missing

**Cluster detection — `cluster_exists()`:**
- `kind get clusters 2>/dev/null | grep -q "^${CKAD_CLUSTER_NAME}$"`

**Cluster health — `cluster_is_healthy()`:**
- Check `kubectl cluster-info --context "${CKAD_KUBE_CONTEXT}" &>/dev/null 2>&1`
- Check all nodes are Ready: count lines from `kubectl get nodes` that don't contain " Ready", must be 0

**Addon install functions (private, prefixed with underscore):**

`_cluster_install_calico()`:
- Use manifest-only method (simpler, per RESEARCH recommendation): `kubectl apply -f "https://raw.githubusercontent.com/projectcalico/calico/${CALICO_VERSION}/manifests/calico.yaml"`
- Wait: `kubectl wait --namespace kube-system --for=condition=ready pod --selector=k8s-app=calico-node --timeout=180s`

`_cluster_install_ingress()`:
- Apply kind-specific manifest: `kubectl apply -f "https://raw.githubusercontent.com/kubernetes/ingress-nginx/controller-${INGRESS_NGINX_VERSION}/deploy/static/provider/kind/deploy.yaml"`
- Wait for admission job first: `kubectl wait --namespace ingress-nginx --for=condition=complete job/ingress-nginx-admission-patch --timeout=60s` (use `job/ingress-nginx-admission-create` if that's the actual job name — check both with `|| true`)
- Wait for controller pod: `kubectl wait --namespace ingress-nginx --for=condition=ready pod --selector=app.kubernetes.io/component=controller --timeout=90s`

`_cluster_install_metrics_server()`:
- Apply: `kubectl apply -f "https://github.com/kubernetes-sigs/metrics-server/releases/download/${METRICS_SERVER_VERSION}/components.yaml"`
- Patch for insecure TLS: `kubectl patch deployment metrics-server --namespace kube-system --type='json' --patch='[{"op": "add", "path": "/spec/template/spec/containers/0/args/-", "value": "--kubelet-insecure-tls"}]'`
- Wait: `kubectl wait --namespace kube-system --for=condition=ready pod --selector=k8s-app=metrics-server --timeout=60s`

**Retry helper — `_cluster_retry_once()`:**
- Takes function name and display name as args
- Prints warning "Retrying {name} install..."
- Calls the function again
- On second failure: prints error and returns 1

**Main lifecycle functions:**

`cluster_start()`:
- Call `cluster_check_deps || return 1`
- If `cluster_exists`: check `cluster_is_healthy` — if healthy, print success "Cluster 'ckad-drill' already running" and return 0. If unhealthy, warn "Cluster unhealthy, recreating..." and call `cluster_stop`.
- Print step-by-step progress (per CONTEXT: each addon step announced):
  - `info "Creating cluster 'ckad-drill'..."`
  - `kind create cluster --name "${CKAD_CLUSTER_NAME}" --config "${CKAD_DRILL_ROOT}/setup/kind-config.yaml"`
  - `info "Installing Calico CNI..."` then `_cluster_install_calico || _cluster_retry_once _cluster_install_calico "Calico" || return 1`
  - `info "Installing ingress-nginx..."` then `_cluster_install_ingress || _cluster_retry_once _cluster_install_ingress "ingress-nginx" || return 1`
  - `info "Installing metrics-server..."` then `_cluster_install_metrics_server || _cluster_retry_once _cluster_install_metrics_server "metrics-server" || return 1`
  - `success "Cluster 'ckad-drill' is ready"`

`cluster_stop()`:
- `info "Deleting cluster 'ckad-drill'..."`
- `kind delete cluster --name "${CKAD_CLUSTER_NAME}" 2>/dev/null || true`
- Clean up stale kubeconfig: `kubectl config delete-context "${CKAD_KUBE_CONTEXT}" 2>/dev/null || true`
- `success "Cluster deleted"`

`cluster_reset()`:
- Call `cluster_check_deps || return 1`
- `info "Resetting cluster..."`
- `cluster_stop`
- `cluster_start`
