**test/unit/cluster.bats** — tests for cluster_check_deps():

Source both common.sh and cluster.sh in setup. The key testing challenge: `cluster_check_deps` calls `command -v` which checks the real system. To test the "missing dep" path, create a wrapper approach:

Approach: Create a test that overrides `command` builtin behavior using bash functions. For each test that needs a missing dep:
```bash
@test "cluster_check_deps fails when kind is missing" {
  # Override command to hide kind
  function kind { return 127; }
  export -f kind
  # Need to also make command -v kind fail
  function command { if [[ "$2" == "kind" ]]; then return 1; else builtin command "$@"; fi; }

  run cluster_check_deps
  assert_failure
  assert_output --partial "kind"
}
```

Alternative simpler approach: Modify PATH to exclude certain tools for testing. Or better — use bats' `PATH` manipulation.

Tests to write:
- `cluster_check_deps succeeds when all deps present` — on the test machine, if all deps happen to be present. If not, skip with `skip "requires all deps installed"`.
- `cluster_check_deps reports all missing deps at once` — mock multiple deps missing, assert output contains ALL of them (not just the first)
- `cluster_check_deps shows install URLs` — assert output contains documentation URLs for missing deps
- `cluster_check_deps checks docker daemon not just binary` — test the docker-daemon-not-running case
- `cluster_exists returns 1 when no cluster` — should work without kind running (kind get clusters returns empty or kind not available)
- `CALICO_VERSION is pinned` — assert variable is set and matches expected pattern
- `INGRESS_NGINX_VERSION is pinned` — assert variable is set
- `METRICS_SERVER_VERSION is pinned` — assert variable is set

Run: `bats test/unit/` should produce passing results. Use `skip` for tests that require specific system state.
