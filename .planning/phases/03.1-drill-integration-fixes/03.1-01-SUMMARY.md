## Decisions Made

- **XDG_CONFIG_HOME for session file control**: `lib/common.sh` unconditionally sets `CKAD_SESSION_FILE="${CKAD_CONFIG_DIR}/session.json"` at source time, overwriting any pre-set env var. Tests must use `XDG_CONFIG_HOME` (which common.sh reads) rather than direct `CKAD_SESSION_FILE` override.
- **Fake kubectl shim over function mocking**: Functions sourced from lib files cannot be overridden by pre-set `export -f` mocks because sourcing redefines them. Instead, a fake `kubectl` executable in `${tmpdir}/bin` is prepended to PATH to record deletion calls.
- **PATH expansion in bash -c strings**: `${PATH}` (unescaped) in a double-quoted `bash -c "..."` string is expanded by the outer shell, embedding the actual PATH. `\${PATH}` (escaped) becomes literal `${PATH}` in single-quoted inner bash context — broken PATH. Use unescaped for correct PATH inheritance.

## Issues Encountered

- **Test mocking approach required iteration**: Initial test design used `export -f` to mock `validator_run_checks` and `scenario_cleanup`, but sourcing lib files after the export redefines the functions. Investigated and switched to: (1) XDG_CONFIG_HOME for session file control, (2) fake kubectl shims for namespace deletion recording, (3) direct bash invocation with proper PATH for _drill_cleanup test.
- **PATH literal in single-quoted bash -c strings**: Using `\${PATH}` in `bash -c "..."` causes the inner bash to receive `${PATH}` in single quotes (literal text, not expanded). This broke jq/rm access in test 19. Fixed by using unescaped `${PATH}` so the outer bats process expands it.
