**BREAK-01 — check) case arm (line ~309):**
Replace the bare `validator_run_checks` call with the set -e safe pattern:
```bash
validator_exit=0
validator_run_checks "${SESSION_SCENARIO_FILE}" "${SESSION_NAMESPACE}" || validator_exit=$?
```
This is the same pattern already used in `_validate_single_scenario` (line ~237). The rest of the check) logic (if/else on validator_exit, elapsed calc, progress_record) remains unchanged. Verify the case arm exits 0 even when validations fail.

**BREAK-02 — next) and skip) case arms (lines ~348-363):**
Bridge SCENARIO_NAMESPACE from SESSION_NAMESPACE before calling scenario_cleanup:
```bash
next)
  session_require
  SCENARIO_NAMESPACE="${SESSION_NAMESPACE}"
  scenario_cleanup
  session_clear
  cluster_check_active || exit "${EXIT_NO_CLUSTER}"
  _drill_start "" "" ""
  ;;

skip)
  session_require
  SCENARIO_NAMESPACE="${SESSION_NAMESPACE}"
  scenario_cleanup
  session_clear
  info "Skipped. Run 'ckad-drill drill' for next scenario."
  ;;
```

**BREAK-03 — _drill_cleanup() function (lines ~29-38):**
Fix the `session_read && scenario_cleanup` chain — separate with `|| true` and bridge the namespace variable:
```bash
_drill_cleanup() {
  trap - INT TERM EXIT
  if [[ -f "${CKAD_SESSION_FILE}" ]]; then
    session_read 2>/dev/null || true
    SCENARIO_NAMESPACE="${SESSION_NAMESPACE:-}"
    scenario_cleanup
    session_clear
  elif [[ -n "${SCENARIO_NAMESPACE:-}" ]]; then
    kubectl delete namespace "${SCENARIO_NAMESPACE}" --ignore-not-found=true 2>/dev/null || true
  fi
}
```

**DEFECT-01 — solution) case arm (lines ~330-338):**
Replace the while-read loop with index-based yq extraction. Extract into `_cmd_solution` helper function (consistent with existing `_cmd_status` and `_cmd_validate_scenario` helpers) so that `local` keyword is valid:
```bash
_cmd_solution() {
  session_require
  header "Solution"
  local step_count step_i step
  step_count=$(yq -r '.solution.steps | length' "${SESSION_SCENARIO_FILE}" 2>/dev/null || echo "0")
  for (( step_i = 0; step_i < step_count; step_i++ )); do
    step=$(yq -r ".solution.steps[${step_i}]" "${SESSION_SCENARIO_FILE}" 2>/dev/null)
    printf '%d. %s\n' "$(( step_i + 1 ))" "${step}"
  done
}
```
Update the `solution)` case arm to call `_cmd_solution`.
