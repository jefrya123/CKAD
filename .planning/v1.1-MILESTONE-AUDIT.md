## Critical Gap: setup.manifest Not Applied (14 Scenarios Broken)

**Root cause:** `lib/scenario.sh` `scenario_setup()` (line 144-157) only applies `setup.manifest` when `setup.commands` contains `"kubectl apply -f -"`. Phase 6 created 14 debug/fix-probe scenarios with manifest-only setup (no `setup.commands`). The manifest is silently skipped.

**Impact:**
- 14 debug/fix scenarios broken in drill mode — broken resources never deployed
- `validate-scenario` gives false PASS — solution creates correct resources without testing broken initial state
- Exam mode may select broken debug scenarios

**Affected (14 scenarios):**

| Domain | File | What's Missing |
|--------|------|----------------|
| 1 | debug-image-pull.yaml | Broken pod never created |
| 1 | debug-volume-mount.yaml | Broken pod never created |
| 2 | debug-api-version.yaml | Broken resource never created |
| 2 | debug-selector-mismatch.yaml | Broken deployment never created |
| 3 | debug-cronjob-schedule.yaml | Broken cronjob never created |
| 3 | debug-probe-port.yaml | Broken pod never created |
| 3 | debug-wrong-image.yaml | Broken pod never created |
| 3 | debug-wrong-port.yaml | Broken pod never created |
| 3 | sc-fix-probe.yaml | Broken pod never created |
| 4 | debug-job-restart-policy.yaml | Broken job never created |
| 4 | debug-missing-secret.yaml | Broken pod never created |
| 5 | debug-label-mismatch.yaml | Broken service never created |
| 5 | debug-multi-bug.yaml | Broken resources never created |
| 5 | debug-netpol-blocks.yaml | Broken netpol never created |

**Fix:** Add standalone manifest application in `scenario_setup()`:
```bash
# After existing setup.commands block, add:
if [[ -z "${setup_commands}" ]]; then
  local setup_manifest
  setup_manifest=$(yq -r '.setup.manifest // empty' "${file}")
  if [[ -n "${setup_manifest}" ]]; then
    echo "${setup_manifest}" | kubectl apply -f - -n "${SCENARIO_NAMESPACE}"
  fi
fi
```
